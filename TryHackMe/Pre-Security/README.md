# Pre-Security: Основы
## Модуль 2. Основы работы сетей (Network Fundamentals)
### Сетевые технологие (networking)
- Узнал, что такое IP и MAC-адрес
- Команда ```ping``` проверяет связь
### Введение в локальные сети (Intro to LAN)
Изучил: 
- **Подсети и IP-адресация**: как делить сеть на части, зачем нужен сетевой адрес, адрес хоста и шлюз по умолчанию.
- **ARP (Address Resolution Protocol)**: как устройства находят MAC-адрес друг друга внутри локальной сети.
- **DHCP (Dynamic Host Configuration Protocol)**: как устройства автоматически получают IP-адрес и другие настройки.
### Модель OSI (OSI Midel)
Изучил модель взаимодействия открытых систем (OSI), 7 уровней и инкапсуляцю.
1) Physical — передача битов по среде (кабель, радиоканал, хаб).
2) Data Link — кадры, MAC-адреса, коммутация в пределах локальной сети (коммутатор).
3) Network — маршрутизация пакетов, IP-адресация (роутер).
4) Transport — надежная или быстрая доставка сегментов, порты (TCP/UDP).
5) Session — управление сеансом связи, синхронизация диалога.
6) Presentation — представление данных (шифрование, сжатие, форматы типа JPEG/ASCII).
7) Application — протоколы прикладного уровня: HTTP, DNS, FTP, SMTP.
### Пакеты и кадры (Packets & Frames)
Главное, что понял:
- Данные передаются не сплошным потоком, а порциями — **Protocol Data Units (PDU)**.
- На каждом уровне модели OSI эта порция называется по-своему.
### Как они связаны с уровнями OSI:
| Уровень OSI | Название PDU | Ключевой адрес |
| :--- | :--- | :--- |
| Transport (4) | **Segment** (Сегмент) | Порт (Port) |
| Network (3) | **Packet** (Пакет) | IP-адрес |
| Data Link (2) | **Frame** (Кадр) | MAC-адрес |

Процесс:
1.  **Инкапсуляция**: Данные сверху вниз обрастают заголовками (Segment -> Packet -> Frame -> биты).
2.  **Декапсуляция**: Снизу вверх заголовки снимаются, пока не останутся чистые данные.
