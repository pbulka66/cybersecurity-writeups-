# Pre-Security: Основы
## Модуль 2. Основы работы сетей (Network Fundamentals)
### Сетевые технологие (networking)
- Узнал, что такое IP и MAC-адрес
- Команда ```ping``` проверяет связь
### Введение в локальные сети (Intro to LAN)
Изучил: 
- **Подсети и IP-адресация**: как делить сеть на части, зачем нужен сетевой адрес, адрес хоста и шлюз по умолчанию.
- **ARP (Address Resolution Protocol)**: как устройства находят MAC-адрес друг друга внутри локальной сети.
- **DHCP (Dynamic Host Configuration Protocol)**: как устройства автоматически получают IP-адрес и другие настройки.
### Модель OSI (OSI Midel)
Изучил модель взаимодействия открытых систем (OSI), 7 уровней и инкапсуляцю.
1) Physical — передача битов по среде (кабель, радиоканал, хаб).
2) Data Link — кадры, MAC-адреса, коммутация в пределах локальной сети (коммутатор).
3) Network — маршрутизация пакетов, IP-адресация (роутер).
4) Transport — надежная или быстрая доставка сегментов, порты (TCP/UDP).
5) Session — управление сеансом связи, синхронизация диалога.
6) Presentation — представление данных (шифрование, сжатие, форматы типа JPEG/ASCII).
7) Application — протоколы прикладного уровня: HTTP, DNS, FTP, SMTP.
### Пакеты и кадры (Packets & Frames)
Главное, что понял:
- Данные передаются не сплошным потоком, а порциями — **Protocol Data Units (PDU)**.
- На каждом уровне модели OSI эта порция называется по-своему.
### Как они связаны с уровнями OSI:
| Уровень OSI | Название PDU | Ключевой адрес |
| :--- | :--- | :--- |
| Transport (4) | **Segment** (Сегмент) | Порт (Port) |
| Network (3) | **Packet** (Пакет) | IP-адрес |
| Data Link (2) | **Frame** (Кадр) | MAC-адрес |

Процесс:
1.  **Инкапсуляция**: Данные сверху вниз обрастают заголовками (Segment -> Packet -> Frame -> биты).
2.  **Декапсуляция**: Снизу вверх заголовки снимаются, пока не останутся чистые данные.

### Расширение вашей сети (Extending Your Network)
Port Forwarding и Firewalls
- **Переадресация портов**: пробрасывает входящий трафик с определенного порта роутера на внутренний компьютер/сервер. Без неё сервер в локальной сети не виден из интернета.
- **Межсетевой экран (Firewall)**: проверяет входящий/исходящий трафик по правилам (адрес, порт, протокол).
  - **Stateful** – следит за состоянием соединения целиком.
  - **Stateless** – проверяет каждый пакет отдельно, не запоминая контекст.

VPN (Virtual Private Network)
Создает зашифрованный туннель между устройствами, делая передачу данных безопасной даже в публичных сетях.
**Основные выгоды VPN**:
- Соединение географически удаленных сетей (офисы).
- Шифрование трафика и защита от перехвата (важно в публичном Wi-Fi).
- Анонимность (журналисты, активисты; но 100% анонимность дает только провайдер без логов).
**VPN-технологии:**
- **PPP (Point-to-Point Protocol)** – протокол прямого соединения. Сам по себе не маршрутизируется (не может покинуть локальную сеть). Использует аутентификацию и шифрование.
- **PPTP** – туннелирует PPP, позволяя ему путешествовать через интернет. Легко настраивается, но шифрование слабее, чем у аналогов.
- **IPSec** – более сложный, но надежный. Шифрует сам IP-трафик на сетевом уровне.
Маршрутизаторы и коммутаторы (Router и Switch)
- **Router** – передает данные между разными сетями, работая на 3-м уровне OSI (Network).
- **Switch** – соединяет устройства в одной сети, обычно на 2-м уровне (Data Link). Многоуровневые коммутаторы могут работать и на 3-м уровне.
## Модуль 3. Как работает интернет (How the Wev Works)
### DNS подробно (DNS in details)
DNS (Domain Name System) — это система, которая переводит удобные для человека имена сайтов (example.com) в IP-адреса (93.184.216.34).
### Иерархия доменов
- **TLD (Top Level Domain)**: .com, .org, .ru, .uk
  - **gTLD** (generic): com, org, net
  - **ccTLD** (country code): ru, uk, de
- **Second-Level Domain**: google, example, yandex — основное имя, которое регистрируется.
- **Subdomain**: www, mail, shop — часть слева от основного имени (до 63 символов).

### Как работает разрешение доменного имени (DNS Resolution)
1. **Кэш клиента**: Проверяется локальный кэш и файл hosts.
2. **Рекурсивный DNS сервер (Recursive Resolver)**: Получает запрос от клиента. Если у него нет ответа, начинает итеративный опрос.
3. **Корневые серверы (Root DNS)**: Перенаправляют рекурсивный сервер к серверам верхнего уровня (TLD) в зависимости от зоны (.com, .org).
4. **Серверы TLD**: Возвращают адреса авторитетных DNS-серверов домена (например, ns1.example.com).
5. **Авторитетный DNS сервер (Authoritative)**: Содержит реальные записи для домена. Отдаёт IP-адрес.
6. Рекурсивный сервер кэширует результат и возвращает клиенту.Общая длина доменного имени не более 253 символов.
### Типы DNS записей
- **A**: IPv4-адрес (93.184.216.34)
- **AAAA**: IPv6-адрес
- **CNAME**: Псевдоним (переадресация на другое имя)
- **MX**: Почтовый сервер (куда слать email)
- **TXT**: Текстовая информация (проверка владения, SPF, DKIM)

### HTTP подробно (HTTP in details)
URL (Uniform Resource Locator)
Структура URL: `схема://хост:порт/путь?строка_запроса#фрагмент`
- **Схема (Scheme)**: протокол (http, https, ftp).
- **Хост (Host)**: доменное имя или IP-адрес сервера.
- **Порт (Port)**: по умолчанию 80 для HTTP, 443 для HTTPS (может быть любой 1-65535).
- **Путь (Path)**: расположение ресурса на сервере.
- **Строка запроса (Query String)**: дополнительные параметры после `?`, например `?id=1&lang=ru`.
- **Фрагмент (Fragment)**: якорь на конкретную секцию страницы после `#`.

HTTP Методы (HTTP Methods)
- **GET** – получение данных (чтение).
- **POST** – отправка данных, создание новой записи.
- **PUT** – полная замена ресурса (обновление).
- **DELETE** – удаление ресурса.

HTTP Status Codes (диапазоны)
- **1xx** – информационные.
- **2xx** – успех (200 OK, 201 Created).
- **3xx** – перенаправления (301 Moved Permanently, 302 Found).
- **4xx** – ошибки клиента (400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 405 Method Not Allowed).
- **5xx** – ошибки сервера (500 Internal Server Error, 503 Service Unavailable).

Заголовки запроса (Request Headers)
- **Host**: указывает, на какой именно сайт запрос (при нескольких сайтах на одном сервере).
- **User-Agent**: информация о браузере/клиенте.
- **Content-Length**: размер тела запроса в байтах.
- **Accept-Encoding**: какие методы сжатия поддерживает клиент.
- **Cookie**: отправляет сохранённые данные сессии обратно на сервер.

Заголовки ответа (Response Headers)
- **Set-Cookie**: сервер просит сохранить куки.
- **Cache-Control**: настройки кэширования.
- **Content-Type**: тип возвращаемых данных (text/html, application/json, image/png).
- **Content-Encoding**: метод сжатия тела ответа.

Cookies
Небольшой фрагмент данных, который сервер через заголовок `Set-Cookie` сохраняет в браузере. При каждом следующем запросе браузер отправляет их обратно в заголовке `Cookie`. Используются для сессий, персонализации, отслеживания.
### Как работают веб-сайты (How Websites Work)
Front End и Back End
- **Front End (Клиентская часть)**: всё, что видит и с чем взаимодействует пользователь в браузере. Создаётся с помощью HTML, CSS, JavaScript.
- **Back End (Серверная часть)**: обработка запросов, логика, работа с базами данных. Находится на сервере и пользователю напрямую не видна.
- **Процесс**: Браузер отправляет HTTP-запрос к серверу → сервер обрабатывает запрос (возможно, обращаясь к БД) → возвращает HTML/CSS/JS → браузер рендерит страницу.

HTML (HyperText Markup Language)
Язык разметки, определяющий структуру веб-страницы.
**Основные элементы (теги):**
- `<!DOCTYPE html>` — декларация типа документа (HTML5), чтобы браузер правильно интерпретировал страницу.
- `<html>` — корневой элемент всей страницы.
- `<head>` — служебная информация (заголовок, кодировка, ссылки на стили).
- `<body>` — видимое содержимое страницы.
- `<h1>`, `<h2>`... — заголовки разных уровней.
- `<p>` — абзац.
- `<button>` — кнопка.
- `<img>` — изображение (атрибут `src` указывает путь к файлу).

**Атрибуты тегов:**
- `class="имя"` — используется для стилизации (CSS), может повторяться у многих элементов.
- `id="уникальное_имя"` — уникальный идентификатор элемента (должен быть один на странице), важен для JavaScript.
- `src="путь"` — путь к ресурсу (для изображений, скриптов).
- Пример: `<p class="bold-text" id="intro">Привет</p>`

**Как посмотреть HTML страницы:**
ПКМ → «Просмотр исходного кода страницы» (View Page Source)

CSS и JavaScript
- **CSS (Cascading Style Sheets)** — отвечает за дизайн: цвета, шрифты, отступы, анимацию. 
- **JavaScript** — язык программирования в браузере, добавляет интерактивность: реакции на клики, валидацию форм, динамическую подгрузку данных.

1. **Раскрытие конфиденциальных данных**: В исходном HTML-коде разработчики часто забывают комментарии, содержащие временные пароли, пути к скрытым директориям, API-ключи. Всегда смотрим исходный код!
2. **Внедрение HTML (HTML Injection)**: Если пользовательский ввод (например, форма комментария) отображается на странице без фильтрации, злоумышленник может вставить свой HTML-код. Это может привести к подделке страницы, краже данных или перенаправлению.

### Собираем все воедино (Putting it all together)
Краткий обзор процесса загрузки сайта
Когда ты вводишь адрес в браузере, происходит следующая цепочка:
1. **DNS-запрос**: компьютер находит IP-адрес веб-сервера по доменному имени.
2. **Установка соединения**: браузер подключается к веб-серверу по протоколу HTTP/HTTPS.
3. **HTTP-взаимодействие**: браузер отправляет HTTP-запрос (метод, заголовки, возможно тело).
4. **Ответ сервера**: веб-сервер возвращает HTML, CSS, JavaScript, изображения и другие ресурсы.
5. **Рендеринг**: браузер собирает страницу и показывает её пользователю.

Инфраструктура вокруг веб-сайта
Для обеспечения надёжности, скорости и безопасности используются дополнительные компоненты.
Балансировщики нагрузки (Load Balancers)
- **Назначение**: равномерно распределяют входящий трафик по нескольким серверам.
- **Резервирование**: если один сервер перестал отвечать, балансировщик исключает его из пула (проверка работоспособности — health check).
- **Алгоритмы распределения**:
  - **Round-robin** — запросы отправляются на серверы по очереди.
  - **Weighted** — запросы распределяются с учётом текущей загрузки серверов (менее загруженный получает запрос).

CDN (Content Delivery Network)
- Сеть серверов, разбросанных по всему миру, которые хранят копии статических файлов (CSS, JS, картинки, видео).
- Пользователь получает данные с ближайшего сервера, что ускоряет загрузку сайта и снижает нагрузку на основной сервер.

Базы данных (Databases)
- Централизованное хранилище информации для веб-сайта (пользователи, посты, товары, логи).
- Серверная часть сайта взаимодействует с базой данных для обработки запросов (например, поиск, добавление записей).

WAF (Web Application Firewall)
- Межсетевой экран уровня веб-приложения, анализирует входящие HTTP-запросы.
- Блокирует вредоносные запросы (SQL-инъекции, XSS, подозрительные паттерны) до того, как они достигнут веб-сервера.
- Работает на 7-м уровне модели OSI, в отличие от обычных файрволов.

Веб-сервер и виртуальные хосты
**Веб-сервер** — это программное обеспечение (Apache, Nginx, IIS), которое:
- Принимает входящие HTTP/HTTPS-запросы.
- Отдаёт клиенту запрошенные файлы или вызывает бэкенд-скрипты для генерации ответа.
- По умолчанию слушает порт 80 (HTTP) и 443 (HTTPS).

**Виртуальные хосты (Virtual Hosts)** — механизм, позволяющий одному веб-серверу обслуживать несколько сайтов с разными доменными именами на одном IP-адресе.
- Сервер смотрит на заголовок `Host` в HTTP-запросе.
- Если домен совпадает с настройками виртуального хоста — отдаёт соответствующий сайт.
- Если домен не указан или не найден — открывает сайт по умолчанию (дефолтный хост).
- Конфигурация хранится в текстовых файлах (например, для Apache: `/etc/apache2/sites-available/`).

Статический и динамический контент
- **Статический контент** — файлы, которые не изменяются сервером и отправляются как есть: изображения, CSS, JS, готовые HTML-страницы.
- **Динамический контент** — генерируется на лету в зависимости от запроса (например, страница профиля, новости, результаты поиска). Часто требует обращения к базе данных.

## Модуль 4. Основы работы с компьютером (Computer Fundamentals)
### Внутри компьютерной системы (Inside a computer System)
Анатомия ПК (сравнение с телом человека)
- **Материнская плата** — Скелет и нервная система: соединяет всё и держит на себе.
- **CPU (Процессор)** — Мозг: обрабатывает инструкции. Современные CPU многоядерные.
- **RAM (ОЗУ)** — Кратковременная память: хранит данные для быстрого доступа. Энергозависима (всё стирается при выключении). Технологии: DDR4/DDR5.
- **HDD / SSD** — Долговременная память:
  - *HDD*: технология с движущимися частями, большая ёмкость, низкая цена, низкая скорость.
  - *SSD*: чипы памяти без движущихся частей, высокая скорость, дороже.
- **GPU (Видеокарта)** — Зрительная кора: получает инфу от ОС и выдает картинку на монитор.
- **Блок питания (PSU)** — Сердце и лёгкие: подает энергию и охлаждает (вентилятор). Если не хватает мощности — система падает.
- **Сетевой адаптер (NIC)** — Органы чувств: проводная (Ethernet) или беспроводная (Wi-Fi) связь.
- **Периферия (Input/Output)** — Рецепторы и мышцы: клавиатуры, мыши, мониторы. Подключаются через USB, HDMI, DisplayPort.

Ключевые разъемы на материнской плате
| Разъем | Компонент | Тип/Форм-фактор |
| :--- | :--- | :--- |
| **CPU Socket** | Процессор | Квадратный, LGA/PGA контакты |
| **DIMM Slots** | Оперативная память (RAM) | Длинные с боковыми защелками |
| **PCIe x16** | Видеокарта (GPU) / Сетевая карта | Длинный слот, усиленный |
| **PCIe x1 / x4** | Сетевые/Звуковые карты | Короткие слоты |
| **24-pin ATX** | Блок питания (PSU) | Главный разъем питания платы |
| **SATA Port** | SSD 2.5" / HDD | L-образный разъем данных |
| **M.2 Slot** | NVMe SSD | Маленький узкий слот прямо на плате |
| **I/O Shield (Задняя панель)** | Периферия (USB, LAN, Audio) | Прямоугольный блок портов |

Последовательность загрузки компьютера (Boot Sequence)
1.  **Power (Нажатие кнопки):** БП подает питание.
2.  **UEFI / BIOS (Прошивка):** Стартует встроенное ПО.
3.  **POST (Power-On Self-Test):** Быстрая проверка железа (CPU, RAM, видео). Если ошибка — писк спикером.
4.  **Select Boot Device:** Поиск устройства для загрузки (SSD, USB-флешка и т.д.).
5.  **Start Bootloader:** Загрузчик запускается и передает управление ядру ОС.

- Загрузка: Power → UEFI/BIOS → POST → Boot Device → OS (ОС в RAM).

### Типы компьютеров (Computer Types)
| Тип | Экран/Клавиатура | Основная цель | Заметки пентестера |
| :--- | :---: | :--- | :--- |
| **Ноутбук (Laptop)** | Да | Портативность для повседневных задач | Часто подключаются к публичным Wi-Fi — цели для сниффинга и MiTM. |
| **Рабочий стол (Desktop)** | Да | Стационарная работа, мощность | Обычно внутри корпоративной сети; взлом через фишинг или физический доступ. |
| **Рабочая станция (Workstation)** | Да | Высокая производительность для проф. задач (CAD, рендер) | Те же векторы, что у десктопа, но часто мощнее и содержит критичные данные (чертежи, коды). |
| **Сервер (Server)** | Нет | Предоставление услуг в сети (веб, БД, почта) | **Главная цель пентестера.** Без клавиатуры/экрана, управляются удалённо (SSH/RDP). Ищем открытые порты и уязвимости сервисов. |
| **Смартфон (Smartphone)** | Да | Мобильная связь, приложения, интернет | Атаки через вредоносные APK, фишинг в мессенджерах, эксплуатация мобильных браузеров. |
| **Планшет (Tablet)** | Да | Потребление контента, заметки | Часто недонастроенная безопасность, используются как «умные» терминалы. |
| **IoT-устройство** | Нет | Выполнение одной специализированной функции (лампочка, камера, замок) | **Золотая жила для взлома.** Слабые пароли, уязвимые прошивки, отсутствие обновлений. Идеальный вход в сеть. |
| **Встраиваемый компьютер (Embedded)** | Нет | Управление другим устройством (автомобиль, мед. оборудование, банкомат) | Критическая безопасность. Часто работают на RTOS, редко обновляются, много багов в firmware. |

### Основы клиент-серверной архитектуры (Client-Server Basics)
Базовые понятия
- **Клиент** — инициатор запроса (браузер, терминал, скрипт). Всегда начинает общение первым.
- **Сервер** — система, предоставляющая услуги (веб-страницы, файлы, почту). Ждёт запросы и отвечает на них.
- **Протокол** — свод правил общения (формат команд, синтаксис, реакции на ошибки). Пример: HTTP.
- **Порт** — числовой идентификатор (1–65535) конкретной службы на сервере.
  - 80 — HTTP
  - 443 — HTTPS
  - 22 — SSH
  - 25 — SMTP
- **DNS** — система, преобразующая доменное имя в IP-адрес (как GPS для сети).

Запрос и ответ
- Клиент формирует запрос по правилам протокола → Сервер обрабатывает → Возвращает ответ.
- При некорректном запросе или недоступном ресурсе сервер возвращает код ошибки (4xx, 5xx).
Основные методы HTTP (9 команд)
`GET` — запросить данные
`POST` — отправить данные (создать)
`PUT` — полностью заменить ресурс
`DELETE` — удалить ресурс
`PATCH` — частично обновить
`HEAD` — только заголовки (без тела ответа)
`OPTIONS` — узнать доступные методы
`CONNECT` — создать туннель
`TRACE` — диагностика (эхо-запрос)
Как смотреть вживую
`F12` → Вкладка **Network (Сеть)** — видны все запросы, заголовки, статусы и ответы.

### Основы виртуализации (Virtualization Basics)
Что такое виртуализация
Технология, позволяющая одному физическому серверу работать как несколько изолированных компьютеров.
- **Аналогия**: Многоквартирный дом. Здание = физический сервер, квартиры = виртуальные машины, управляющий = гипервизор.
Гипервизор
Специальное ПО, создающее и управляющее виртуальными машинами (ВМ).
- Делит физические ресурсы (CPU, RAM, диск) между ВМ.
- Обеспечивает изоляцию и безопасность.
- Управляет жизненным циклом ВМ (запуск, остановка, клонирование).

Типы гипервизоров
1. **Тип 1 (bare-metal)** — работает прямо на железе, без хостовой ОС.
   - Быстрый, эффективный, для серверов и дата-центров.
   - Примеры: VMware ESXi, Proxmox VE, Microsoft Hyper-V.
2. **Тип 2 (hosted)** — устанавливается как приложение внутри существующей ОС.
   - Проще в установке, идеален для обучения и тестирования.
   - Примеры: VirtualBox, VMware Workstation.

Виртуальные машины (ВМ) vs Контейнеры
- **ВМ**: Полноценный виртуальный компьютер со своей ОС, ядром, драйверами. Изолированы на уровне ядра. Запускаются дольше, потребляют больше ресурсов.
- **Контейнеры**: Изолированные процессы, использующие ядро хостовой ОС. Легковесные, запускаются мгновенно.
  - Упаковывают приложение и все его зависимости.
  - Работают одинаково на любой машине.
  - Образ (Image) — неизменяемый шаблон для создания контейнера (как рецепт).

Ключевые термины
- **Виртуализация**
- **Гипервизор**
- **Виртуальная машина (VM)**
- **Контейнер**
- **Образ контейнера**
- **Сетевые порты** (контейнеры/ВМ используют их для связи)

Преимущества виртуализации
- Экономия средств и эффективное использование ресурсов
- Безопасное тестирование и изоляция (критично для пентеста!)
- Быстрое развертывание, портативность и масштабируемость
- Централизованное управление
- Гибкость и удобство создания лабораторий

### Основы облачных вычислений (Cloud Computing Fundamentals)
Что такое облачные вычисления
Доступ к вычислительным ресурсам (серверы, хранилище, сеть) через интернет вместо локального компьютера. Позволяет приложениям быть доступными глобально, надёжными и легко масштабироваться.
Ключевые преимущества
- **Масштабируемость** – ресурсы легко увеличить или уменьшить под нагрузку.
- **Самообслуживание по требованию** – создание/удаление серверов и хранилищ в пару кликов.
- **Оплата за использование (pay-as-you-go)** – плата только за потреблённые ресурсы.
- **Безопасность** – провайдер защищает физическую инфраструктуру и предлагает инструменты безопасности.
- **Высокая доступность** – приложения продолжают работать даже при сбоях оборудования.
- **Глобальный доступ** – ресурсы доступны из любой точки мира.

Типы развёртывания
- **Публичное облако** – инфраструктура провайдера доступна через интернет всем. Дешёво, масштабируемо, не требует управления железом. Примеры: AWS, Azure, Google Cloud. Подходит стартапам и веб-приложениям.
- **Частное облако** – инфраструктура только для одной организации. Полный контроль, кастомизация, соблюдение строгих норм (банки, госорганы, здравоохранение).
- **Гибридное облако** – комбинация публичного и частного. Чувствительные данные остаются в частном облаке, а при пиковых нагрузках задействуется публичное (электронная коммерция в сезон распродаж).

Модели облачных услуг (Service Models)
- **IaaS (Infrastructure as a Service)** – аренда виртуальных серверов, хранилища, сетей. Вы управляете ОС и приложениями, провайдер – физическим железом.
  - *Пример:* Amazon EC2, виртуальные машины в Azure.
- **PaaS (Platform as a Service)** – готовая среда для разработки и запуска приложений без управления ОС и инфраструктурой. Вы только пишете код и деплоите.
  - *Пример:* Heroku, Google App Engine.
- **SaaS (Software as a Service)** – готовое приложение, доступное через браузер. Провайдер управляет всем. Вы просто пользуетесь.
  - *Пример:* Gmail, Zoom, Microsoft 365.

Важные термины
- **Публичное / Частное / Гибридное облако** – типы развёртывания.
- **IaaS, PaaS, SaaS** – уровни облачных услуг по степени управления.
- **Amazon EC2** – один из самых популярных сервисов IaaS (виртуальные серверы в облаке AWS).

## Модуль 5. Основы операционных систем (Operating System Basics)
### Операционные системы: Введение (Operating Systems: Introduction
Уровни системных привилегий
- **Пространство ядра (Kernel Space)** — привилегированная область. Ядро ОС имеет полный доступ к CPU, памяти, устройствам. Управляет системными вызовами.
- **Пользовательское пространство (User Space)** — здесь работают приложения. Прямой доступ к железу запрещён, только через системные вызовы к ядру. Обеспечивает изоляцию и стабильность.
Безопасность и аутентификация
- **Аутентификация** — проверка личности (логин, пароль, биометрия).
- **Права доступа** — что разрешено читать/писать/выполнять пользователю или процессу.
- **Изоляция** — каждый процесс в своей защищённой среде.
- **Защита системы** — системные файлы защищены от несанкционированных изменений.
Интерфейсы ОС
- **GUI (Graphical User Interface)** — окна, значки, меню. Взаимодействие кликами/касаниями.
- **CLI (Command Line Interface)** — текстовый ввод команд. Точное и быстрое управление.
Типы операционных систем
- **Десктопные** — ПК и ноутбуки: богатый GUI, многозадачность (Windows, macOS, Linux).
- **Серверные** — веб-хостинг, БД, облака: без GUI или минимальный, высокая доступность, удалённое администрирование.
- **Мобильные** — смартфоны, планшеты: сенсорный интерфейс, энергоэффективность, песочницы приложений (iOS, Android).
- **Встраиваемые** — IoT, автомобили, роутеры: минималистичные, работа на ограниченном железе.
- **Виртуальные / Облачные** — виртуальные машины, контейнеры, облачные инстансы: лёгкие, масштабируемые, быстрое развёртывание.

### Оснвоы Windows (Windows Basics)
Типы учётных записей
- **Гость (Guest)** — временный доступ, минимум прав, нельзя менять настройки. (Обычно отключена).
- **Стандартный (Standard User)** — повседневная работа: запуск приложений, личные настройки. Не может устанавливать софт или менять системные параметры.
- **Администратор (Administrator)** — полный контроль: установка ПО, управление пользователями, изменение конфигурации системы. Важно: встроенная учётная запись *Administrator* (RID 500) обычно скрыта, административные права даются через членство в группе *Administrators*.

Основные элементы интерфейса
- **Рабочий стол** — основное рабочее пространство с файлами, папками и ярлыками.
- **Панель задач** — доступ к приложениям, уведомлениям и системному трею.
- **Меню «Пуск»** — вход в приложения, настройки, выключение/перезагрузка.
- **Поиск** — мгновенный поиск приложений, настроек и файлов.
- **Проводник (File Explorer)** — управление файлами и папками.

Настройка системы
- **Параметры Windows (Settings)** — современный центр управления системой, устройствами, персонализацией и безопасностью.
- **Панель управления (Control Panel)** — устаревший, но всё ещё нужный интерфейс для тонких административных задач (например, управление сертификатами, старые апплеты).

Инструменты безопасности и мониторинга
- **Диспетчер задач (Task Manager)** — мониторинг процессов, производительности, автозагрузки в реальном времени.
- **Безопасность Windows (Windows Security)** — единая панель управления встроенной защитой: антивирус, файрвол, контроль учётных записей.
- **Microsoft Defender Antivirus** — встроенный антивирус, защита от вредоносного ПО в реальном времени.
- **Windows Defender Firewall** — встроенный межсетевой экран, фильтрует входящий/исходящий сетевой трафик по правилам.

Обновления и приложения
- **Центр обновления Windows (Windows Update)** — поддерживает систему, драйверы и защиту в актуальном состоянии.
- **Microsoft Store** — официальный магазин для установки проверенных приложений.

### Основы работы с командной строкой Windows (Windows CLI Basics)
Основные команды
| Команда | Описание |
| :--- | :--- |
| `cd <путь>` | Сменить директорию (Change Directory). |
| `dir` | Список файлов и папок в текущей директории. |
| `dir /a` | Показать все файлы, включая скрытые и системные. |
| `dir /s` | Рекурсивный поиск: показать файлы в текущей папке и всех подпапках. |
| `type <файл>` | Вывести содержимое текстового файла в консоль (аналог `cat` в Linux). |

Полезные уточнения
- `dir /a` выводит файлы с атрибутами **H** (скрытые) и **S** (системные), которые не видны при обычном `dir`.
- `systeminfo` — аналог `uname -a` и части `df` из Linux, даёт кучу данных для разведки: какая версия Windows, какие патчи установлены, сколько памяти.
- `ipconfig` — чтобы увидеть больше деталей (DNS-суффикс, MAC-адреса, DHCP), используй `ipconfig /all`.

Что важно запомнить

- Ключи (опции) начинаются с `/` (slash), в отличие от `-` или `--` в Linux.
- Автодополнение по `Tab` работает и здесь — пользуйся.
- Для справки по любой команде используй `<команда> /?`.

### Безопасность операционной системы (Operating System Security)

Как взламывают системы (базовые угрозы)
- **Перебор паролей (Brute-force / Dictionary attack)** — автоматический подбор логина и пароля. Защита: сложные пароли, блокировка после N попыток.
- **Троянские кони (Trojans)** — вредонос, маскирующийся под легальное ПО. Даёт злоумышленнику бэкдор в систему.
- **Программы-вымогатели (Ransomware)** — шифруют файлы и требуют выкуп. Опасны для организаций.
- **Повышение привилегий** — эксплуатация уязвимостей для получения прав root/Administrator.

Практика: доступ к системе через SSH
С помощью AttackBox подключались к удалённому серверу и исследовали его изнутри.

| Команда | Что делали |
| :--- | :--- |
| `ssh USERNAME@10.114.134.195` | Подключение к серверу по SSH под указанной учётной записью. |
| `whoami` | Проверка, под кем мы в системе. |
| `ls` | Список файлов. |
| `cat FILENAME` | Чтение содержимого файла (например, паролей или заметок). |
| `history` | Просмотр истории команд текущего пользователя (можно найти чувствительные данные). |
| `su - name` | Переключение на другого пользователя (например, root) при знании пароля. |

Учётные записи с полным доступом
- **Linux, Android, Apple (macOS/iOS)** — `root`. Полный неограниченный контроль над системой.
- **Microsoft Windows** — `Administrator` (встроенная учётная запись или член группы Administrators). Те же полные права.

Получив такие права, злоумышленник может:
- Устанавливать/удалять программы
- Читать/изменять любые файлы
- Создавать новых пользователей
- Скрывать следы

## Модуль 6. Основы программного обеспечения (Software Basics)
### Представление данных (Data Representation)

Бит и байт
- **Бит (bit)** — двоичная цифра, принимает значение `0` или `1`.
- **Байт (byte / октет)** — группа из 8 бит. Один байт может представить 256 разных значений (0–255).
- 8 бит = 1 байт.

Системы счисления
| Система | Основание | Цифры | Особенность |
| :--- | :---: | :--- | :--- |
| Двоичная (Binary) | 2 | 0, 1 | Базовый язык компьютеров. |
| Восьмеричная (Octal) | 8 | 0–7 | Группировка по 3 бита. Используется реже. |
| Десятичная (Decimal) | 10 | 0–9 | Повседневная система. |
| Шестнадцатеричная (Hex) | 16 | 0–9, A–F | 4 бита = одна hex-цифра, один байт = две hex-цифры. |

Представление цвета
- Цвет в компьютере — это комбинация **красного (R), зелёного (G) и синего (B)**.
- На каждый канал выделяется **1 байт** (8 бит), итого 24 бита (3 байта) на пиксель.
- Каждый канал может принимать значение от 0 до 255 (256 градаций).
- **Шестнадцатеричная запись цвета**: `#RRGGBB`, например `#FF0000` — красный, `#00FF00` — зелёный, `#0000FF` — синий.
- Всего доступно **16 777 216** цветов (256³).

### Кодирование данных
Представление vs Кодирование
- **Представление данных** — способ физического хранения в виде битов и байтов.
- **Кодирование данных** — набор правил, сопоставляющих числа (байты) с осмысленными символами (буквами, цифрами, знаками).

Основные стандарты кодирования
- **ASCII (American Standard Code for Information Interchange)**
  - 7-битная кодировка (значения 0–127).
  - Включает английские буквы, цифры, знаки препинания и управляющие символы.
  - Основа всех современных кодировок.
- **ISO-8859**
  - 8-битное семейство кодировок (256 символов), расширяющее ASCII.
  - Разные части для разных языков: Latin-1 (западноевропейские), Latin-2 (центрально- и восточноевропейские) и т.д.
  - Недостаток: нельзя использовать символы из разных частей в одном файле.
- **Unicode**
  - Универсальный стандарт, присваивающий уникальный номер (кодовую точку) каждому символу всех письменностей мира.
  - Решает проблему смешивания языков и несовместимости кодировок.
  - Самые популярные кодировки семейства Unicode: UTF-8 и UTF-16.

### Python: Простая демонстрация (Python: simple demo)
Разбор программы "Угадай число"
Код реализует базовую логику: компьютер загадывает случайное число от 1 до 20, а пользователь угадывает, получая подсказки «слишком мало», «слишком много», «вне диапазона». После успеха выводится количество попыток.

Ключевые компоненты Python в коде
- **Импорт модулей (`import random`)** — подключаем готовые инструменты. `random.randint(1, 20)` генерирует секретное число.
- **Переменные (`secret`, `tries`, `guess`)** — именованные контейнеры для хранения чисел и счётчиков.
- **Ввод и вывод (`print()`, `input()`)** — общение с пользователем. `input()` всегда возвращает строку.
- **Преобразование типов (`int(text)`)** — превращаем строку в целое число, чтобы можно было сравнивать с `secret`.
- **Цикл `while guess != secret:`** — повторяет блок кода, пока условие истинно (число не угадано). Основа автоматизации перебора.
- **Условный оператор `if / elif / else`** — проверяет условия и выполняет нужный блок. Ветка `else` срабатывает, когда число угадано.

### JavaScript: простая демонстрация (JavaScript% simple demo)

Разбор программы "Угадай число" на JavaScript
Та же логика, что в Python-примере, но реализованная на JS с использованием среды Node.js. Игрок угадывает число от 1 до 20, программа даёт подсказки и считает попытки.

Ключевые особенности JavaScript в коде
- **Импорт модулей Node.js** — для ввода/вывода используются `node:readline/promises` и `node:process`. В браузере бы использовался `prompt()`.
- **Асинхронный ввод** — `await rl.question()` позволяет не блокировать выполнение других операций (важно для веб-приложений).
- **Строгие операторы** — `===` и `!==` без приведения типов, чтобы избежать багов.
- **Объявление переменных**:
  - `const` — для неизменяемых значений (MIN_NUMBER, rl).
  - `let` — для изменяемых (guess, tries).
